あなたはフルスタック実装AIです。以下の仕様に基づき、Webサービスを実装してください。最速リリースが目的です。

# 0. 名前

- 公開名：スワコレ（環境変数 NEXT_PUBLIC_APP_NAME で変更可能）

- 内部コードネーム：swacole

- ドメイン：swacole.com

# 1. サービス目的

コレクターアイテム（ガチャガチャ・一番くじ・推し活グッズ等）でダブった・不要になったアイテム同士を、金銭取引なしの物々交換で安全にマッチング・取引できる場を提供する。



# 2. スコープ（MVP）

- 対象：コレクターアイテム全般（カテゴリ：ガチャガチャ / 一番くじ / 推し活グッズ / その他）

- 交換：1対1のみ

- マッチング：検索＋交換提案が軸（自動マッチは後）

- 配送のみ、送料は発送者負担（双方が自分の発送分を負担）

- 本人性：電話番号認証必須

- 安全：取引ステータス、取引ルーム内メッセージ、通報/ブロック、相互評価、紛争（証拠提出→運営判断）



# 3. アイテム登録方式（ユーザー写真ベース + タグ分類）

- アイテム登録はユーザーが現物を撮影し、タグで分類する方式

- カテゴリ選択（必須）：ガチャガチャ / 一番くじ / 推し活グッズ / その他

- 作品名タグ（必須）：サジェスト付き自由入力（例：鬼滅の刃、推しの子、ワンピース）

- キャラ・アイテム名タグ（任意）：サジェスト付き自由入力（例：竈門炭治郎、A賞 フィギュア）

- タグ辞書（旧カタログDBを再活用）：作品名・キャラ名のサジェスト用マスターデータ。運営が追加・管理

- ユーザーが新しい作品名を入力した場合はそのまま保存（運営が後からタグ辞書に反映）



# 4. コレクション

- 「持っている」アイテムを登録できる（コレクション）

- 各アイテムに flags: is_public, is_tradeable

- デフォルト is_public=false

- is_tradeable=true の場合は is_public=true を強制（交換成立のため）

- 公開範囲：ログインユーザーのみ

- ユーザー詳細に「交換に出してる」「コレクション（公開）」の2タブ



# 5. アイテム登録要件

- 写真4枚必須（ユーザーが現物を撮影）

- カテゴリ選択（必須）：ガチャガチャ / 一番くじ / 推し活グッズ / その他

- 作品名タグ（必須・サジェスト付き）

- キャラ・アイテム名タグ（任意・サジェスト付き）

- 状態：未開封/開封済/傷あり（最小）

- 数量（整数、デフォルト1）

- メモ（任意）

- 交換可/公開トグル



# 6. 画面（MVP）

- トップ（説明/導線）

- 検索（キーワード、状態フィルタ）

- アイテム詳細（写真、状態、出品者信頼、交換提案CTA、通報/ブロック）

- ユーザー詳細（評価、取引回数、SMS済バッジ、2タブ）

- マイアイテム（登録/編集/削除、取引中は削除不可）

- WANT（欲しい）登録/一覧（選択式、自由入力なし）

- 交換提案作成（相手アイテム固定＋自分の出すアイテム選択＋任意メッセージ）

- 提案一覧（受信/送信）

- 取引ルーム（ステータス、メッセージ、住所入力、発送報告、受取確認、評価、通報/紛争）

- マイページ（設定、ヘルプ）

- 運営画面（通報/紛争/ユーザー制限・凍結/カタログ反映/申請処理）



# 7. 取引ステータス

PROPOSED, ACCEPTED, ADDRESS_PENDING, ADDRESS_LOCKED, SHIPMENT_PENDING, SHIPPED, RECEIVED, COMPLETED, DISPUTE, CANCELLED



# 8. 期限（決め打ち）

- 住所入力期限：成立から48h

- 発送期限：住所確定から5日

- 受取確認期限：到着見込みから3日

期限超過時は紛争提起が可能になり、DISPUTEに遷移できる（裁定は運営判断）



# 9. 住所（PII）方針：暗号化して保持

- 住所は取引成立後に双方が入力し、相手に表示できる

- 住所はDBに平文で保存しない。暗号化して保存する（application-layer encryption を前提とする）

- 復号できるのは当該取引の当事者のみ。運営は原則復号できない方針（必要なら例外条件を別途定義）

- 取引完了後、住所は自動削除（推奨：完了後7日以内）



# 10. 追跡番号

- optionalフィールドとして発送報告に追跡番号入力欄は置く（任意）



# 11. 通報・紛争

- 通報対象：ユーザー/アイテム/取引

- 紛争理由：未発送、状態違い、未着

- 証拠：画像添付

- 運営の裁定結果：警告、一定期間制限、凍結、取引キャンセル/完了の強制



# 12. ペナルティ（決め打ち）

- 未発送で有責裁定：1回警告、2回7日制限、3回凍結

- 詐欺/虚偽が濃厚：即凍結



# 13. ブロック

- ブロック相手からの提案は不可

- 進行中取引はブロックしても取引ルームは維持



# 14. 受け入れ条件（E2E）

- A/Bが電話認証→コレクション登録→検索→提案→成立→住所入力→発送報告→受取→評価まで完走できる

- 通報→運営裁定→ユーザー制限/凍結が動く

- 期限超過で紛争提起できる

- 住所が暗号化されて保存され、完了後に削除される
